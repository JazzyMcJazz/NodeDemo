NPM

What is NPM?

NPM (originally short for Node Package Manager) is a package manager for JavaScript and the world's largest software registry. NPM comes with a CLI which grants the user access to many useful features (see below).


Usage

Developers can use NPM to install tools globally in their development environment to help assist in development. Locally (per independent project) NPM is used to install and manage packages (define dependencies). NPM also includes build tools and a way to define metadata.


Common Commands

npm init

Starts a wizard that initializes a new project with the current directory as root. At the conclusion of the wizard a package.json file is created.


npm install

Installs or updates dependencies found in package.json


npm install <package_name>

Adds the package to the list of dependencies found in package.json and installs it


package.json

An example of a package.json file:

{
  "name": "project_name",
  "version": "1.0.0",
  "description": "describes the project",
  "main": "app.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "start": "node app.js"
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/SomeUser/SomeRepo.git"
  },
  "keywords": [
    "npm",
    "express",
  ],
  "author": "John Smith",
  "license": "MIT",
  "dependencies": {
    "express": "^4.17.3"
  }
}

Scripts

You can write scripts that can be called by NPM with a custom keyword by adding them under scripts in your package.json file.

Common examples are scripts for testing and executing your project:

"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1",
  "start": "node app.js"
},

Environment

In Node.js environment variables can be accessed with process.env


You can define environment variables in the command you use to run your node project
For example:

PORT=3000 node app.js

NOTE:
The above example only works in a bash terminal. If you are using Windows CMD or PowerShell the syntax is different. A way to get around this in a cross-OS development environment is to install the package cross-env.
It is recommended you install cross-env as a development dependency:

npm install --save-dev cross-env

Your package.json should now include this:

"devDependencies": {
  "cross-env": "^7.0.3"
}

Now you can define environment variables using a single syntax across any terminal

cross-env PORT=3000 node app.js
