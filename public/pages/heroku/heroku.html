Heroku

What is Heroku?

Heroku is a Cloud Platform as a Service which makes deploying and maintaining web applications easy. Setting up a CI/CD pipeline for automatic deployment is also possible


How to Deploy a web app on Heroku?

Before you can deploy a Node.js project on Heroku you need to have two things in order:


First, you need to have a start script in your package.json. Heroku uses this script to run node projects

"scripts": {
  "start": "node app.js"
},

Second, because Heroku assigns a dynamic port through the environment you need to set your application to listen on process.env.PORT. In this example a fallback port has been set as well, which is practical in development

const PORT = process.env.PORT || 3000;
app.listen(PORT)

Now we are ready to deploy. Go to heroku.com and register/login


Once you are logged in, in the top-right corner click New > Create new app





One the next page you specify your apps name (it has to be unique) and a region and click Create App





Once you are on your new apps page go to Deploy tab





You have different options regarding how you get your source code onto Heroku. The easiest way is to use GitHub and connect your repository to your Heroku app





Once you have found your repo and connected to it, scroll down to Automatic deploys or Manual deploy - depending on your preference - choose your desired branch to deploy and hit Deploy Branch





Congratulations! Your app is now live in the cloud



NOTE

This guide only works for projects with no external dependencies like for example databases
